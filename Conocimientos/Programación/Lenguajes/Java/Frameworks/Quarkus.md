Optimizado por Red Hat para microservicios y serverless. Ofrece tiempos de arranque extremadamente rápidos y un consumo de memoria muy bajo (subatómico) al compilar a binarios nativos con GraalVM.
Es ideal para entornos cloud native, permitiendo combinar modelos de programación reactivos e imperativos.

**Características Clave de Quarkus:**

- **Supersónico y Subatómico:** Diseñado para un inicio rápido y bajo uso de recursos, ideal para contenedores.
- **Desarrollo Eficiente:** Ofrece recarga en vivo (live coding) para una retroalimentación rápida.
- **Compatibilidad:** Soporta estándares como Jakarta EE y MicroProfile.
- **Compilación Nativa:** Utiliza GraalVM para crear ejecutables independientes.
- **Integración:** Compatible con herramientas como Maven, Gradle y los principales IDEs (VS Code, IntelliJ)
- Puede generar proxies estáticas, a diferencia de las [Proxies dinámicas] de Springboot, haciéndolo más ligero en runtime.
