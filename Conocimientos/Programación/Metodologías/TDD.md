(Test Driven Develpment)

Metodología de programación en la cual primero escribes los tests y después el código para esos tests.

El ciclo de TDD es rojo -> verde -> refactorizar

- Rojo: Se escribe un test que va a fallar ya que aun no existe el código a probar.
- Verde: Se escribe el código mínimo necesario para que el test pase sin fallar.
- Refactorizar: Se mejora el código optimizándolo y limpiándolo sin romper los tests.


En resumen es una metodología para mejorar la calidad del código, reduciendo la posibilidad de bugs y mejorando el diseño pero puede ser bastante mas lento.
